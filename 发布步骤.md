# 发布到 npm / GitHub 你需要做的事

按顺序完成以下步骤即可把 `resizable-pro-table` 发布到 npm，并在 GitHub 上自动发布。

---

## 第一步：安装依赖并本地构建

在项目根目录或 `resizable-pro-table` 目录下执行：

```bash
cd resizable-pro-table
npm install
npm run build
```

确认 `dist/` 下生成：`index.js`、`index.mjs`、`index.d.ts`、`style.css`。若有报错，先解决再继续。

---

## 第二步：确认 npm 包名可用

1. 打开 https://www.npmjs.com/
2. 搜索 `resizable-pro-table`
3. 若已被占用，在 `resizable-pro-table/package.json` 里把 `"name"` 改成其他名字（如 `@你的用户名/resizable-pro-table`）。若使用 scoped 包（带 `@xxx/`），发布时需加 `--access public`。

---

## 第三步：登录 npm（未登录时）

在终端执行：

```bash
npm login
```

按提示输入 npm 用户名、密码、邮箱。若使用镜像（如淘宝），发布前建议切回官方源：

```bash
npm config set registry https://registry.npmjs.org/
```

---

## 第四步：创建 GitHub 仓库并推送

1. 在 GitHub 新建一个**空仓库**（例如 `resizable-pro-table`），不要勾选 “Add a README”。
2. 在**本仓库**里，只把 `resizable-pro-table` 作为独立项目推送上去。两种方式二选一：

**方式 A：只推送子目录为新仓库（推荐）**

在项目根目录执行（把 `你的用户名` 和 `resizable-pro-table` 换成你的 GitHub 用户名和仓库名）：

```bash
cd resizable-pro-table
git init
git add .
git commit -m "chore: initial resizable-pro-table library"
git remote add origin https://github.com/你的用户名/resizable-pro-table.git
git branch -M main
git push -u origin main
```

**方式 B：整个当前仓库就是 resizable-pro-table**

若你打算整个项目就是一个库（不再保留 ant-table-column-resize 业务代码），则直接在根目录 `git remote add origin ...` 并 push 即可。

---

## 第五步：配置自动发布（GitHub Actions）

1. 在 GitHub 仓库页面：**Settings → Secrets and variables → Actions**。
2. 点击 **New repository secret**：
   - Name: `NPM_TOKEN`
   - Value: 在 https://www.npmjs.com/settings/你的用户名/tokens 新建 **Automation** 类型 Token，复制粘贴保存。

3. 在仓库中创建文件 `.github/workflows/release.yml`（若用方式 A，该文件放在你 push 上去的 `resizable-pro-table` 项目根目录），内容如下：

```yaml
name: Release and Publish to npm

on:
  release:
    types: [published]

jobs:
  publish:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          registry-url: 'https://registry.npmjs.org'

      - run: npm ci
      - run: npm run build

      - run: npm publish --access public
        env:
          NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}
```

说明：

- 触发条件：在 GitHub 仓库里 **创建 Release（发布）** 时自动执行。
- `--access public`：若包名是 **scoped**（如 `@xxx/resizable-pro-table`）必须加；普通包加了也无妨。

---

## 第六步：发布版本

1. 在 `resizable-pro-table/package.json` 里确认 `version`（如 `1.0.0`）。
2. 提交并推送（若用方式 A，在 `resizable-pro-table` 目录下）：

   ```bash
   git add .
   git commit -m "chore: prepare 1.0.0"
   git push
   ```

3. 在 GitHub 仓库页面：**Releases → Create a new release**：
   - 选择或新建 Tag，例如 `v1.0.0`
   - Release title 可填 `v1.0.0`
   - 点击 **Publish release**

4. 打开 **Actions** 页，应看到 “Release and Publish to npm” 工作流运行；成功后到 https://www.npmjs.com/package/resizable-pro-table 即可看到新包（或你改过的包名）。

---

## 之后每次发版

1. 在 `package.json` 里把 `version` 改为新版本（如 `1.0.1`）。
2. 提交并 push。
3. 在 GitHub 再建一个 Release，Tag 填 `v1.0.1`（与 version 一致），发布后 Actions 会自动执行并发布到 npm。

---

## 在当前业务项目里使用已发布的包

在 `ant-table-column-resize` 项目中：

```bash
npm install resizable-pro-table
```

然后修改页面中的引用：

```tsx
// 原来
import ResizableProTable from '@/components/ResizableProTable';

// 改为
import ResizableProTable from 'resizable-pro-table';
import 'resizable-pro-table/dist/style.css';
```

确认无引用后，可删除 `src/components/ResizableProTable` 目录。
